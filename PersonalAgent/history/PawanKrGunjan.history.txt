+## Core idea
+
+- **ReAct agent**: “Think and do” pattern where the model alternates between reasoning (“Thought”) and taking actions with tools/environments (“Action → Observation”) until it can answer. [apxml](https://apxml.com/courses/langchain-production-llm/chapter-2-sophisticated-agents-tools/agent-architectures)
+- **Reflection agent (Reflexion)**: “Think about your own thinking” pattern where the model generates an answer, critiques it, and then revises or uses that critique as memory for future attempts. [linkedin](https://www.linkedin.com/posts/pallavi-saxena-09181b12b_react-vs-reflection-how-ai-agents-think-activity-7365825935377387520-AKdi)
+
+## Key differences
+
+| Aspect                  | ReAct agent                                         | Reflection / Reflexion agent                                    |
+|-------------------------|-----------------------------------------------------|------------------------------------------------------------------|
+| Main goal              | Solve a task by reasoning + tool use now. [apxml](https://apxml.com/courses/langchain-production-llm/chapter-2-sophisticated-agents-tools/agent-architectures) | Improve quality by self‑evaluation and correction. [linkedin](https://www.linkedin.com/posts/pallavi-saxena-09181b12b_react-vs-reflection-how-ai-agents-think-activity-7365825935377387520-AKdi) |
+| Loop structure         | Thought → Action → Observation → (repeat) → Answer. [apxml](https://apxml.com/courses/langchain-production-llm/chapter-2-sophisticated-agents-tools/agent-architectures) | Answer → Critique/feedback → Revised answer and/or stored reflection. [linkedin](https://www.linkedin.com/posts/pallavi-saxena-09181b12b_react-vs-reflection-how-ai-agents-think-activity-7365825935377387520-AKdi) |
+|Planning vs learning    | Strong on on‑the‑fly planning, weak on learning from past mistakes. [aiprotocolshub](https://aiprotocolshub.com/blog/react-vs-reflexion) | Explicitly designed to learn from previous failures or feedback. [aiprotocolshub](https://aiprotocolshub.com/blog/react-vs-reflexion) |
+|Memory use              | Often uses short‑term context; no special self‑memory. [apxml](https://apxml.com/courses/langchain-production-llm/chapter-2-sophisticated-agents-tools/agent-architectures) | Stores “reflections” as episodic memory to guide future attempts. [emergentmind](https://www.emergentmind.com/topics/reflection-agent) |
+|Performance trade‑offs  | Faster, cheaper; may give confident but wrong answers with no self‑check. [aiprotocolshub](https://aiprotocolshub.com/blog/react-vs-reflexion) | Slower, more tokens; usually higher accuracy and robustness on complex tasks. [aiprotocolshub](https://aiprotocolshub.com/blog/react-vs-reflexion) |
+|Typical use cases       | Tool‑calling agents (web, DB, code), dynamic workflows, when you mostly need correct first shot. [apxml](https://apxml.com/courses/langchain-production-llm/chapter-2-sophisticated-agents-tools/agent-architectures) | Coding with debug cycles, long‑horizon tasks, high‑stakes reasoning where errors are costly. [aiprotocolshub](https://aiprotocolshub.com/blog/react-vs-reflexion) |
+
+## How they often work together
+
+Many practical systems use ReAct for interacting with tools and environment, and add a Reflection module on top to review trajectories or answers and either retry or update a memory of “what to avoid next time.” [promptingguide](https://www.promptingguide.ai/techniques/reflexion)
